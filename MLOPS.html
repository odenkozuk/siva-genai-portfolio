// ML Algorithms · Basics · Use Cases
01What is the difference between Supervised, Unsupervised, and Reinforcement Learning?
Easy▶
Supervised: Model learns from labeled data (input → output). Goal: predict output for new inputs.
Unsupervised: No labels. Model finds patterns/clusters in data.
Reinforcement: Agent learns via rewards/penalties through environment interaction.
Examples
  • Supervised → Spam detection, house price prediction, fraud detection
  • Unsupervised → Customer segmentation (K-Means), topic modeling (LDA)
  • Reinforcement → AlphaGo, self-driving cars, ChatGPT RLHF fine-tuning
02Explain Linear Regression. When would you use it?
Easy▶
Models relationship between a continuous target variable and one or more features by fitting a straight line: y = mx + b. Minimizes Mean Squared Error (MSE) via Ordinary Least Squares or Gradient Descent.
Use When
  • Target is continuous (not categorical)
  • Linear relationship between features and target
  • Interpretability matters (e.g., finance, economics)
Real Examples
Sales forecasting, predicting insurance premiums, house price estimation
Assumptions
Linearity, No multicollinearity, Homoscedasticity, Normally distributed residuals
03What is Logistic Regression and how is it different from Linear Regression?
Easy▶
Logistic Regression is used for binary/multi-class classification. It applies the sigmoid function to output a probability [0,1].
σ(z) = 1 / (1 + e^(-z)) → outputs P(class=1)
Key Differences
  • Linear Reg → continuous output; Logistic → probability → class label
  • Loss: Linear uses MSE; Logistic uses Binary Cross-Entropy
  • Linear uses OLS; Logistic uses MLE or Gradient Descent
Use Cases
Email spam detection, disease diagnosis (cancer/no cancer), churn prediction
04What is a Decision Tree? Explain how it splits nodes.
Easy▶
A tree-like model that splits data based on feature thresholds to make predictions. Each internal node = a feature test; leaf = prediction.
Split Criteria
  • Gini Impurity (used by CART) – measures how often a randomly chosen element would be misclassified
  • Entropy / Information Gain – based on reduction in uncertainty (ID3, C4.5)
  • MSE reduction – for regression trees
Pros / Cons
✅ Interpretable, handles non-linear relationships, no scaling needed
❌ Prone to overfitting; unstable with small data changes
Use Cases
Credit risk scoring, medical diagnosis rules, customer churn
05Random Forest vs Gradient Boosting — key differences?
Medium▶
Both are ensemble methods using decision trees.
Random Forest (Bagging)
  • Builds N trees in parallel on random data+feature subsets
  • Aggregates by majority vote / averaging
  • Reduces variance (overfitting)
  • Robust to noise, fast to train
Gradient Boosting (Boosting)
  • Builds trees sequentially; each corrects errors of previous
  • Minimizes loss via gradient descent on residuals
  • Reduces bias; higher accuracy but slower, prone to overfitting
  • Popular implementations: XGBoost, LightGBM, CatBoost
When to use
RF → quick baseline, noisy data | GB → Kaggle competitions, tabular data where accuracy matters
06What is SVM? Explain kernel trick.
Medium▶
Support Vector Machine finds the optimal hyperplane that maximizes the margin between classes. Support vectors = data points closest to the hyperplane.
Kernel Trick
Maps data to higher-dimensional space without computing coordinates explicitly, making non-linearly separable data linearly separable.
  • Linear kernel – linearly separable data
  • RBF (Radial Basis Function) – most common; good for non-linear boundaries
  • Polynomial kernel – for polynomial decision boundaries
Use Cases
Text classification (high-dim), image recognition, bioinformatics (gene classification)
07K-Means Clustering — how does it work?
Easy▶
Unsupervised algorithm that groups N data points into K clusters by minimizing intra-cluster variance (WCSS).
Algorithm Steps
  1. Initialize K centroids randomly (or K-Means++)
  2. Assign each point to nearest centroid (Euclidean distance)
  3. Recompute centroids as mean of cluster points
  4. Repeat steps 2-3 until convergence
Choosing K
Elbow Method – plot WCSS vs K, pick elbow point | Silhouette Score – measures cluster cohesion
Use Cases
Customer segmentation, document clustering, anomaly detection, image compression
08What is overfitting and underfitting? How do you fix them?
Easy▶
Overfitting: Model memorizes training data → high train accuracy, poor test accuracy. High variance.
Underfitting: Model too simple → poor on both train and test. High bias.
Fix Overfitting
  • Regularization: L1 (Lasso), L2 (Ridge), Dropout (NN)
  • More training data, data augmentation
  • Cross-validation, early stopping
  • Reduce model complexity (fewer features, shallower tree)
Fix Underfitting
Add more features, increase model complexity, train longer, reduce regularization
09What are Precision, Recall, F1-Score and when to use which?
Easy▶
Precision = TP / (TP + FP) → "Of predicted positives, how many are correct?" Recall = TP / (TP + FN) → "Of actual positives, how many did we catch?" F1-Score = 2 * (P * R) / (P + R) → Harmonic mean
When to prioritize
  • Precision → Spam detection (don't want legit mail in spam), fraud alerts (reduce false alarms)
  • Recall → Cancer detection (don't miss actual cases), safety-critical systems
  • F1 → Imbalanced datasets where both matter (NER, document classification)
  • AUC-ROC → When you need threshold-independent performance measure
10Explain Bias-Variance Tradeoff
Medium▶
Total Error = Bias² + Variance + Irreducible Noise
Bias = Error from wrong assumptions (underfitting). Simple model → high bias.
Variance = Error from sensitivity to training data (overfitting). Complex model → high variance.
Tradeoff
Increasing model complexity reduces bias but increases variance. Goal: find the sweet spot.
Ensemble methods (Bagging ↓ variance, Boosting ↓ bias) help manage this tradeoff.
11What is cross-validation? Types?
Easy▶
Technique to evaluate model generalization by partitioning data into train/validation splits multiple times.
Types
  • K-Fold CV – Split into K folds; train on K-1, validate on 1, rotate. Most common.
  • Stratified K-Fold – Preserves class distribution per fold. Use for imbalanced data.
  • LOOCV – Leave-One-Out; K = N. Computationally expensive.
  • Time-Series Split – Forward-chaining to prevent data leakage in temporal data.
12What is PCA and why is it used?
Medium▶
Principal Component Analysis is a dimensionality reduction technique. It transforms features into a new set of orthogonal axes (principal components) ordered by variance explained.
Steps
Standardize → Compute covariance matrix → Eigen decomposition → Select top K eigenvectors → Project data
Use Cases
  • Reduce dimensionality before ML (speeds up training)
  • Remove multicollinearity
  • Visualize high-dimensional data (2D/3D plot)
  • Facial recognition (Eigenfaces)
Limitation
Components are linear combinations → hard to interpret | Not suitable when variance ≠ importance
13What is feature engineering? Common techniques?
Easy▶
Process of using domain knowledge to create, transform, or select features that improve model performance.
Techniques
  • Encoding – One-hot, Label, Target encoding for categoricals
  • Scaling – StandardScaler (z-score), MinMaxScaler, RobustScaler
  • Binning – Age groups, income brackets
  • Interaction features – Multiply two features (area = length × width)
  • Date/time features – Day of week, hour, is_weekend
  • Handling missing values – Mean/median imputation, KNN impute, indicator flag
14Explain L1 vs L2 Regularization
Medium▶
Both add a penalty term to the loss function to prevent overfitting by constraining model weights.
L1 (Lasso): Loss + λ * Σ|wᵢ| → Sparse weights (some = 0) L2 (Ridge): Loss + λ * Σwᵢ² → Small weights but rarely zero ElasticNet: Combination of L1 + L2
When to use
  • L1 – Feature selection built-in; when many irrelevant features exist
  • L2 – When all features matter; correlated features
  • ElasticNet – Large datasets with correlated features
15What is the difference between Bagging and Boosting?
Medium▶
Both combine multiple weak learners but differ in approach:
Bagging (Bootstrap Aggregating)
Parallel training on random subsets → reduces variance. Example: Random Forest
Boosting
Sequential training, each model fixes errors of previous → reduces bias. Examples: XGBoost, AdaBoost, LightGBM
Stacking
Train diverse models, use a meta-learner on their predictions. Best for competitions.
// Deep Learning · Neural Networks · Architectures
01How does backpropagation work?
Medium▶
Algorithm to compute gradients of loss w.r.t. each weight using the chain rule, then update weights via gradient descent.
Steps
  1. Forward pass – compute predictions and loss
  2. Backward pass – propagate gradients layer by layer using chain rule
  3. Weight update – w = w - α * ∂L/∂w
Vanishing Gradient Problem
Gradients shrink exponentially in deep networks with sigmoid/tanh. Fix: ReLU, batch norm, skip connections (ResNet), LSTM gates
02ReLU vs Sigmoid vs Tanh — when to use?
Easy▶
Sigmoid: σ(x) = 1/(1+e^-x) → [0,1] | Binary output layer Tanh: tanh(x) → [-1,1] | RNNs, hidden layers (zero-centered) ReLU: max(0, x) → [0,∞) | Default hidden layer activation (faster) LeakyReLU: max(0.01x, x) → fixes dying ReLU GELU/Swish → used in Transformers (BERT, GPT)
Dying ReLU
Neurons outputting 0 for all inputs → no gradient → stuck. Fix: LeakyReLU, ELU, or lower learning rate
03What is Batch Normalization and why use it?
Medium▶
Normalizes layer inputs to zero mean and unit variance per mini-batch during training. Adds learnable γ (scale) and β (shift).
Benefits
  • Reduces internal covariate shift → faster convergence
  • Acts as regularizer (reduces need for Dropout)
  • Allows higher learning rates
  • Reduces sensitivity to weight initialization
Alternatives
Layer Norm (used in Transformers, NLP) | Instance Norm (style transfer) | Group Norm
04CNN Architecture — how do convolutions work?
Medium▶
Convolutional layers apply learned filters (kernels) across the input to detect local patterns (edges, textures, shapes).
Key Operations
  • Conv Layer – Feature extraction via dot product of filter + input patch
  • Pooling – Spatial downsampling (MaxPool, AvgPool)
  • Padding – SAME (preserve size) | VALID (shrinks)
  • Stride – Controls step size of filter movement
Famous Architectures
LeNet → AlexNet → VGG → ResNet (skip connections) → EfficientNet → ViT
Use Cases
Image classification, object detection (YOLO), segmentation, OCR
05What are RNNs, LSTMs, and GRUs?
Medium▶
RNN: Processes sequences by maintaining hidden state. Suffers from vanishing gradients on long sequences.
LSTM: Adds cell state + 3 gates (input, forget, output) to control information flow over long sequences.
GRU: Simplified LSTM with 2 gates (reset, update). Fewer params, similar performance.
Use Cases
Time-series forecasting, text generation, speech recognition, machine translation (pre-Transformer era)
Why Transformers replaced them
Sequential processing → slow; Transformers parallelize with attention, handle longer contexts better
06Optimizers: SGD vs Adam vs RMSProp
Medium▶
SGD (Stochastic Gradient Descent)
Updates weights per mini-batch. Simple, generalizes well, but slow convergence, sensitive to LR.
Adam (Adaptive Moment Estimation)
Combines momentum + RMSProp. Maintains per-parameter LR. Fast convergence, default choice for most tasks.
RMSProp
Adapts LR per parameter using moving average of squared gradients. Good for RNNs.
When to use
Adam → default | SGD + momentum → CV tasks (often better final accuracy) | AdamW → LLM fine-tuning (decoupled weight decay)

01What is MLOps and why does it matter?
Easy▶
MLOps (ML + DevOps) is a set of practices to reliably deploy, monitor, and maintain ML models in production at scale. Bridges the gap between data scientists and operations.
Core Pillars
  • Data versioning & pipeline automation
  • Model versioning, experiment tracking
  • CI/CD for ML (model testing, validation gates)
  • Model serving (REST APIs, batch inference)
  • Monitoring (data drift, model drift, latency)
Tools
MLflow, DVC, Kubeflow, Vertex AI, Azure ML, SageMaker, Weights & Biases, Evidently AI
02What is model drift? Types and how to detect?
Medium▶
Degradation in model performance over time due to changes in real-world data patterns.
Types
  • Data Drift (Covariate Shift) – Input feature distribution changes. P(X) changes, P(Y|X) same. Example: User demographics shift.
  • Concept Drift – Relationship between input and output changes. P(Y|X) changes. Example: Fraud patterns evolve.
  • Label Drift – Target variable distribution changes.
  • Prediction Drift – Model output distribution shifts.
Detection Methods
PSI (Population Stability Index), KS-Test, Chi-Squared test, Evidently AI, custom statistical tests
Response
Retrain model, update feature pipeline, add new data sources, trigger alerts
03Explain ML Pipeline stages
Easy▶
Data Ingestion → Data Validation → Data Preprocessing → Feature Engineering → Model Training → Model Evaluation → Model Registry → Deployment → Monitoring
Key Concepts
  • Data Versioning – DVC, Delta Lake; track dataset changes
  • Experiment Tracking – MLflow, W&B; log params, metrics, artifacts
  • Model Registry – Centralized store with versioning and staging (Staging → Production)
  • Feature Store – Reusable, consistent features for train & serve (Feast, Tecton)
04Model deployment strategies — Blue/Green, Canary, Shadow
Medium▶
  • Blue/Green – Run old (blue) and new (green) models simultaneously. Switch 100% traffic at once. Easy rollback.
  • Canary – Gradually shift traffic (5% → 25% → 100%) to new model. Monitor each stage.
  • Shadow/Dark Launch – New model receives real traffic but responses are discarded. Compare outputs silently without impacting users.
  • A/B Testing – Split traffic between models; measure business metrics (CTR, conversion).
  • Multi-armed Bandit – Dynamic traffic allocation based on real-time performance.
05What is data leakage and how to prevent it?
Medium▶
Occurs when information from outside the training set (future data or target info) leaks into the model, causing inflated performance metrics that don't generalize.
Common Sources
  • Scaling/normalization computed on full dataset before split
  • Feature derived from target (e.g., including label in features)
  • Time-series: using future data to predict past
  • Duplicate rows split across train/test
Prevention
Always split data FIRST, then fit transformers only on train set. Use pipelines (sklearn Pipeline). Careful temporal splitting for time-series.
06What is model explainability? SHAP vs LIME?
Medium▶
XAI (Explainable AI) – Techniques to understand why a model made a prediction.
SHAP (SHapley Additive exPlanations)
Based on game theory Shapley values. Assigns each feature a contribution to the prediction. Consistent and globally accurate. Supports summary plots, waterfall plots, beeswarm plots.
LIME (Local Interpretable Model-agnostic Explanations)
Creates a local linear approximation around a prediction. Faster but less stable; explanations can vary with different random seeds.
Other Tools
Integrated Gradients (DL), Attention visualization (Transformers), Partial Dependence Plots
07What is CI/CD for ML? How is it different from software CI/CD?
Medium▶
In addition to code, ML CI/CD must also version and test data + models.
ML CI/CD Pipeline
  • CI – Data validation tests, unit tests for feature transforms, model training on sample data, metric threshold gates
  • CD – Auto-deploy model if metrics pass, canary rollout, rollback on regression
  • CT (Continuous Training) – Retrain trigger on data drift or schedule; unique to ML
Tools
GitHub Actions + MLflow | Azure DevOps + Azure ML | Kubeflow Pipelines | SageMaker Pipelines
08Batch inference vs Real-time (online) inference
Easy▶
Batch Inference
Run predictions on large datasets periodically (daily, weekly). Cheaper, scalable. Examples: Churn prediction overnight, product recommendation refresh.
Real-time Inference
Low-latency predictions per request (ms). Deployed as REST API (FastAPI, Flask, TorchServe). Examples: Fraud detection at transaction time, chatbots, recommendation on page load.
Tradeoffs
Batch → cost-efficient, stale predictions | Real-time → fresh, costly, needs infra (load balancer, auto-scaling)
// NLP Basics · Models · Use Cases
01What is tokenization? Types?
Easy▶
Process of breaking text into smaller units (tokens) for processing.
Types
  • Word tokenization – Split by whitespace/punctuation. Simple but large vocab, OOV problem.
  • Subword tokenization – BPE (GPT), WordPiece (BERT), SentencePiece (T5). Handles OOV, widely used in LLMs.
  • Character tokenization – Each char is a token. Small vocab but long sequences.
BPE (Byte-Pair Encoding)
Iteratively merges most frequent character pairs. "unhappiness" → ["un", "happy", "ness"]. Used by GPT-2/3/4.
02TF-IDF vs Word2Vec vs BERT embeddings
Medium▶
TF-IDF (Term Frequency – Inverse Document Frequency)
Sparse vector; word importance = frequency in doc × rarity in corpus. No semantics. Good for keyword-based search, BM25.
Word2Vec (2013)
Dense 300-dim vectors via CBOW/Skip-gram. Captures semantic similarity. Static embeddings (same vector regardless of context). "bank" (river) = "bank" (finance).
BERT Embeddings (Contextual)
Transformer-based. Each word gets a different embedding based on full sentence context. "bank" is disambiguated. State-of-the-art for downstream NLP tasks.
Use Cases
TF-IDF → IR, document retrieval | Word2Vec → similarity, clustering | BERT → QA, NER, classification, RAG
03Explain the Transformer architecture (Attention Is All You Need)
Hard▶
Introduced in 2017 by Vaswani et al. Replaced RNNs as the dominant architecture for NLP.
Key Components
  • Multi-Head Self-Attention – Each token attends to all others; captures long-range dependencies. Multiple heads capture different relationships.
  • Positional Encoding – Sine/cosine vectors added to embeddings since Transformer has no inherent order.
  • Feed-Forward Network – Two linear layers per position (independent).
  • Encoder-Decoder – Encoder (BERT) processes input; Decoder (GPT) generates output autoregressively.
  • Layer Norm + Residual connections – Stabilize training.
Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V
Variants
Encoder-only: BERT (classification, NER) | Decoder-only: GPT (generation) | Encoder-Decoder: T5, BART (translation, summarization)
04What is BERT? How does it differ from GPT?
Medium▶
BERT (Bidirectional Encoder Representations from Transformers)
  • Encoder-only Transformer. Pre-trained on MLM (Masked Language Modeling) + NSP.
  • Bidirectional: sees full context (left + right).
  • Best for: Classification, NER, QA (extractive), sentence similarity.
GPT (Generative Pre-trained Transformer)
  • Decoder-only Transformer. Pre-trained on CLM (Causal/Autoregressive language modeling).
  • Unidirectional: only sees past tokens.
  • Best for: Text generation, summarization, code generation, chat.
Use Cases
BERT → sentiment analysis, document classification, search ranking | GPT → chatbots, copilots, content generation
05What is RAG (Retrieval-Augmented Generation)?
Medium▶
Architecture that combines a vector retrieval system with an LLM to generate grounded, up-to-date answers without retraining.
Pipeline
User Query → Embed query → Vector Search (FAISS/Pinecone/Weaviate) → Retrieve top-K chunks → Inject into LLM prompt → Generate answer
Why RAG over Fine-tuning?
  • No expensive retraining; knowledge updated by updating vector DB
  • Reduces hallucination by grounding in retrieved context
  • Supports real-time or private knowledge bases
Use Cases
Enterprise knowledge chatbots, legal document Q&A, internal IT helpdesk, customer support
06What is fine-tuning? LoRA / PEFT?
Hard▶
Fine-tuning adapts a pre-trained LLM to a specific task/domain by continuing training on task-specific labeled data.
Full Fine-Tuning
Update all weights. Expensive (needs GPUs), risk of catastrophic forgetting.
LoRA (Low-Rank Adaptation) — PEFT
Freezes base model. Adds small trainable low-rank weight matrices (ΔW = A × B) to attention layers. Reduces trainable params by ~1000x.
Used in QLoRA (quantized), PEFT library (HuggingFace).
Other PEFT Methods
Prefix Tuning, Prompt Tuning, Adapter layers, IA³
Use Cases
Domain-specific chatbots, code assistants, medical/legal LLMs, custom tone/style
07Named Entity Recognition (NER) — how does it work?
Easy▶
Sequence labeling task: classify each token into entity categories (PERSON, ORG, LOC, DATE, etc.).
Approaches
  • Rule-based – Regex, dictionaries. Fast but brittle.
  • CRF (Conditional Random Field) – Classic ML approach; models label dependencies.
  • BiLSTM-CRF – Deep learning with sequential context.
  • Transformer-based – BERT + token classification head. State of the art. spaCy transformer models.
Labeling Scheme
BIO tagging: B-ORG (Beginning), I-ORG (Inside), O (Outside)
Use Cases
Resume parsing, invoice extraction, news indexing, medical record analysis
08What is Prompt Engineering? Key techniques?
Easy▶
Craft of designing inputs to LLMs to elicit desired outputs without changing model weights.
Key Techniques
  • Zero-shot – No examples; rely on model's pre-training knowledge
  • Few-shot – Include 2-5 input-output examples in prompt
  • Chain-of-Thought (CoT) – "Let's think step by step" → improves reasoning
  • Self-consistency – Generate multiple CoT paths, majority vote
  • ReAct – Reasoning + Acting; model reasons and takes tool actions
  • Structured output prompting – Ask for JSON/XML format
System Prompt Best Practices
Define role clearly | Specify output format | Add constraints and examples | Use delimiters (XML tags, ```)
09What are Hallucinations in LLMs? How to mitigate?
Medium▶
LLMs generate plausible-sounding but factually incorrect or unsupported content.
Types
  • Intrinsic – Contradicts source material
  • Extrinsic – Cannot be verified from source
Mitigation Strategies
  • RAG – Ground output in retrieved factual documents
  • RLHF – Align model with human preference feedback
  • Self-consistency – Sample multiple responses, pick consistent one
  • Constrained decoding – Limit vocabulary/output format
  • Fact verification layer – Post-process with fact-checker model
  • Temperature = 0 – For deterministic factual tasks
10What is semantic search vs keyword search?
Easy▶
Keyword Search (BM25, TF-IDF)
Matches exact words. Fast, interpretable but misses synonyms and intent. "car" ≠ "automobile".
Semantic Search (Dense Retrieval)
Embeds query and documents into vector space, finds nearest neighbors by cosine similarity. Understands meaning not just keywords.
Hybrid Search
Combines both (RRF – Reciprocal Rank Fusion). Best of both worlds. Used in production RAG systems.
Embedding Models
text-embedding-ada-002 (OpenAI), BAAI/bge, E5, Cohere Embed, sentence-transformers
